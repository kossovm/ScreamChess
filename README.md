# Scream Chess 🧠♟️

🔗 Live: [scream-chess.vercel.app](https://scream-chess.vercel.app/)

Шахматы — это не только ходы на доске. Это то, как человек думает под давлением. Scream Chess превращает обычную партию в инструмент **поведенческого ассессмента**: каждое колебание пальца, каждая дрожь голоса, каждый импульсивный размен — это сигнал. Платформа собирает их в реальном времени и склеивает в «психологический паспорт» игрока.

Полезно для трёх аудиторий:
- **Игроку** — узнать о себе то, что не покажет рейтинг ELO.
- **HR/рекрутеру** — за 5 минут получить срез того, как кандидат принимает решения.
- **Тренеру** — увидеть, где ученик «ломается» под темпом.

## ✨ Что есть в проекте

### 🎮 Режимы игры

- **Локальная партия** — двое за одним устройством. Сразу два психо-паспорта, один на каждого.
- **Против Stockfish** — 11 уровней сложности, движок крутится в Web Worker.
- **Онлайн-комната** — по ссылке, через Supabase Realtime, со встроенным голос-чатом WebRTC.
- **Найти соперника** — матчмейкинг рейтинговых игр одним кликом.
- **Открытое лобби** — список рейтинговых комнат со ставками. Можно создать свою или подключиться к чужой.
- **Режим «Интервью»** — для HR. 5-минутная партия с расширенной поведенческой панелью только у хоста.
- **Анализатор партии** — импорт PGN, вычисление оценки Stockfish, разбор от AI-тренера.

### 🎤 Голосовой ввод (полностью офлайн)

- Локальная распознавалка речи через **Vosk WebAssembly**. Модели русского и английского лежат прямо в `public/vosk-models/`, грузятся один раз и кешируются в IndexedDB.
- Альтернатива Web Speech API, который завязан на Google и часто блокируется в РФ/КЗ.
- AudioWorklet-пайплайн (а не deprecated ScriptProcessor) — стабильно работает даже под COEP `require-corp`, который у нас включён ради Stockfish.
- Парсер ходов толерантен к мисхирам Vosk: «**едва четыре**» → `e2 e4`, «джи один я три» → `g1 a3`, и т.п.
- Команды: ходы (`е два е четыре`), рокировки (`короткая/длинная рокировка`), отмена (`отмена/откати`), сдача (`сдаюсь`), ничья (`ничья`).
- Хоткей `V` — везде, не теряя фокус.

### 📞 Голос-чат в онлайне

- Чистый WebRTC через STUN-серверы Google.
- Initiator-pattern: белый создаёт offer, чёрный — answer. Хендшейк через `voice-hello` на случай race condition.
- Включается автоматически в режиме интервью.

### 🤖 AI-тренер

- Stockfish прогоняет каждую позицию (~200ms на ход), считает потерю в сантипешках, классифицирует ход (best/good/inaccuracy/mistake/blunder).
- **Gemini получает структурированные данные движка** и обязан цитировать ходы дословно — никаких выдуманных вариантов. Жёсткий промпт.
- Разбор партии в 4 секциях: что было хорошо · где сломалось · следующий ход · психология.
- На главной локали (RU/EN). Поддерживает оба языка одновременно — нужный промпт выбирается по локали клиента.

### 🧠 Психологический паспорт

- 5 шкал (1-10): толерантность к риску, стресс, импульсивность, терпение, адаптивность.
- Считается **локально**, без внешних запросов: время на ход, частота взятий/шахов, голосовая интонация.
- В режиме «vs2» (локальная партия) строится **отдельно для белых и чёрных** — больше не сваливается в одну кучу.
- В коде: `src/lib/psychoAnalyzer.ts` — открытая эвристика, всё прозрачно.

### 💼 Режим «Интервью» (для HR)

- Хост видит закрытую панель с live-метриками кандидата:
  - сколько импульсивных ходов (<2.5 с) и глубоких (>15 с)
  - средне-арифметика времени, % озвученных ходов
  - последняя сказанная вслух фраза
  - визуальный 5-минутный таймер
- 💡 **Подсказки от движка прямо в панели** — Stockfish считает лучший ход для хоста, чтобы рекрутер не тратил силы на шахматы и сосредоточился на интервью. Можно отключить чекбоксом.
- Кнопка «Сформировать HR-отчёт» → Gemini генерирует поведенческий разбор в 4-х абзацах: стиль решений · реакция на давление · профиль риска · контекст использования. Без диагнозов и «нанимать/не нанимать» — только наблюдения.

### ⭐ Рейтинг и монеты

- У каждого юзера: `rating` (стартует с 10) и `coins` (стартует с 5000).
- Win: +2 рейтинга. Draw: +1. Loss: −1. Resign: −2. Рейтинг не уходит в минус.
- Ставки: 0/50/100/250/500/1000 монет. Победитель забирает ставку, проигравший платит.
- Пополнение баланса: единоразовая «милостыня» 1000 монет, если на счету меньше 1000. Кнопка в профиле.
- Все апдейты атомарно через Supabase RPC `apply_match_result` с idempotency-таблицей `rated_results` (одна партия = одно начисление).

### ⏱️ Контроль времени и устойчивость

- Часы 1/3/5/10/30 мин или без часов. Хост (белый) выставляет, оба видят синхронно через broadcast.
- Падение флага → автоматический результат + рейтинговая корректировка.
- **30 секунд офлайна → автоматическая дисквалификация**. Жёлтый баннер с обратным отсчётом.
- **Refresh-устойчивость**: позиция, история ходов, стартовое время, флаг Ready — всё persists в `localStorage`. Перезагрузка не сбивает партию.
- **State-sync на reconnect**: только что зашедший клиент шлёт `state-request`, оппонент отвечает `state-snapshot` с массивом SAN-ходов. Доска восстанавливается мгновенно.

### 👤 Профиль

- Имя, фамилия, никнейм, город — всё редактируется inline.
- Рейтинг, баланс монет, количество партий, когнитивный стиль.
- Локализация (RU/EN) с автодетектом по `navigator.language`.

### 💬 Двуязычный интерфейс

- Полностью переключается RU ⇄ EN кнопкой в шапке.
- AI-тренер, психо-паспорт, HR-отчёт — всё на выбранном языке.
- Voice-engine тоже автоматически берёт `ru-RU` или `en-US` модель.

### 💰 Стартаперский лендинг

- Hero, режимы, фичи, **Enterprise-блок** (3 use-case'а: HR-скрининг, оценка лидеров, API & embed).
- 3 тарифа: Hobby (Free) · Team ($49/мес) · Enterprise (Custom).
- Контакт для демо.

## 🚀 Локальный запуск

```bash
npm install
cp .env.local.example .env.local        # заполнить ключи
npm run setup:vosk                       # скачать модели речи (~85 МБ)
npm run dev                              # http://localhost:3000
```

### Env-переменные (`.env.local`)

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_...   # или legacy NEXT_PUBLIC_SUPABASE_ANON_KEY
GEMINI_API_KEY=AIza...
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Supabase setup

В SQL Editor выполнить по очереди:

1. `supabase/schema.sql`
2. `supabase/migrations/001_economy.sql` — рейтинг, монеты, charity
3. `supabase/migrations/002_open_lobbies.sql` — публичные комнаты
4. `supabase/migrations/003_signup_fix.sql` — фикс ошибок регистрации
5. `supabase/migrations/004_lobby_update_policy.sql` — RLS update для lobby
6. `supabase/migrations/005_signup_robust.sql` — exception-handler для триггера
7. `supabase/migrations/006_username_and_names.sql` — username из email + first/last name

### Vosk-модели

`npm run setup:vosk` скачает русскую и английскую модели с alphacephei.com, переупакует в `.tar.gz` и положит в `public/vosk-models/`. Модели в git'е (~85 МБ, чтобы Vercel мог их раздавать как статику).

Если хочешь хостить их где-то ещё — поставь `NEXT_PUBLIC_VOSK_MODEL_RU` и `NEXT_PUBLIC_VOSK_MODEL_EN` в `.env.local`.

## 🚧 Что не сделано (известные ограничения)

- Disconnect-абуз в рейтинговых партиях: если белый отвалился до конца, рейтинг не пересчитывается (только белый сторона вызывает RPC). Чёрный делает это вручную через 30-секундный таймер дисквалификации, но если оба офлайн — не запишется.
- Матчмейкинг — простой first-come-first-served, без подбора по близости рейтинга.
- Чёрные не могут изменить контроль времени — только хост (белый).
- Нет push-уведомлений / e-mail при матче.
